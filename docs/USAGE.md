# 使用说明

---

## 1. 快速开始

1. 在侧边栏点击 **新建实例**，输入实例名称
2. 切换到 **连接设置** Tab，填写以下必填项：
   - **API Key** — 你的 DeepSeek API 密钥(兼容所有OpenAI格式)
   - **API URL** — API 请求地址（默认 `https://api.deepseek.com/v1/chat/completions`）
   - **WS URL** — Minecraft 服务器 WebSocket 地址（默认 `ws://127.0.0.1:8080`）
3. 切换到 **机器人设置**，确认 **游戏内名称** 与游戏中的角色名一致
4. 点击顶部工具栏的 **启动** 按钮
5. 在游戏中 @机器人 即可开始对话

> 第一次使用？建议先创建一个测试实例，用默认配置启动验证连接是否正常。

---

## 2. 基础配置说明

| 配置项 | 说明 |
|---|---|
| `bot_name` | 机器人在游戏内的玩家名。用于自身回音检测和消息展示 |
| `admin_username` | 管理员的游戏 ID。启用了 `admin_only` 的自定义指令只有该玩家可使用 |
| `model_pro` | Pro 模型名（默认 `deepseek-v4-pro`），适合复杂推理场景 |
| `model_flash` | Flash 模型名（默认 `deepseek-v4-flash`），适合快速响应场景 |
| `ws_url` | Minecraft 服务器的 WebSocket 地址 |

> `bot_name` 必须与游戏内的实际玩家名完全一致，否则自身回音检测会失效。

## 3. 触发与发送规则

### 唤醒前缀（`trigger_prefix`）
游戏中发送包含此前缀的消息才会触发机器人回复。默认为 `@bot`。设为空则所有消息都会触发。

### 发送者识别规则（`sender_patterns`）
逗号分隔的模板，用于从聊天消息中提取发言者名字。默认 `<{name}>,{name}:`：
- `<{name}>` — 匹配 `Steve` 格式
- `{name}:` — 匹配 `Steve:` 格式

### 冷却时间（`cooldown_seconds`）
每个玩家触发回复后的冷却时间（秒）。冷却期内该玩家再次触发会收到"请稍后再试"提示。

### 发送模式（`send_mode`）

| 模式 | 说明 |
|---|---|
| **me** | 以 `/me` 动作形式发送，显示为 `* bot_name 消息` |
| **raw** | 直接发送原生文本（默认推荐） |
| **say** | 以 `/say` 形式发送（已弃用） |
| **json** | 以 JSON 协议格式发送（需要服务端插件支持） |

### 前缀配置
- `reply_prefix` — 回复消息的前缀文字
- `comment_prefix` — 氛围评论的前缀文字
- `system_msg_prefix` — 系统消息（如冷却提示）的前缀文字
- `skip_msg_prefix` — 逗号分隔，匹配这些前缀的聊天消息将被完全忽略

### 发送字节上限（`max_command_bytes`）
单条消息的最大字节数。超长回复会被自动截断，截断时会在末尾追加 `…` 标记。
如果截断后仍超长，消息会被拆分为多条依次发送。

## 4. AI 参数

| 参数 | 说明 | 推荐值 |
|---|---|---|
| `temperature` | 生成随机性，越高越有创意 | 0.7 |
| `max_tokens` | 单次回复最大 Token 数 | 300 |
| `max_history` | 对话记忆条数（最近 N 条消息） | 20 |
| `global_context_size` | 氛围评论的最小上下文消息数 | 10 |
| `api_timeout` | API 调用超时时间（秒） | 15 |
| `retry_count` | API 调用失败后的重试次数 | 2 |
| `retry_delay` | 重试间隔（秒） | 1.5 |

> `temperature` 值越高（最大 2），回复越随机和有创造力；值越低（最小 0），回复越确定和保守。

---

## 5. 连接稳定性

| 参数 | 说明 | 推荐值 |
|---|---|---|
| `heartbeat_check_interval` | 心跳检查间隔（秒） | 30 |
| `heartbeat_timeout` | 心跳超时时间（秒）。超过此时间未收到任何消息则主动断开重连 | 600 |
| `reconnect_delay` | 普通重连延迟（秒） | 5 |
| `reconnect_delay_long` | 长时间重连延迟（秒），用于连接被拒绝等场景 | 10 |

> 如果你的服务器消息频率较低，可以适当增大 `heartbeat_timeout` 避免误断连。

## 6. 提示词与自定义指令

### 系统提示词（`system_prompt`）
设定机器人的角色和性格。每次 AI 调用都会附带此提示词。如果配置了 `bot_name`，系统会在提示词前自动注入一段身份声明。

### 氛围评论提示词（`auto_comment_prompt`）
用于氛围评论的专用提示词。留空则使用默认提示词。

### 氛围评论间隔（`auto_comment_interval`）
自动触发氛围评论的间隔时间（秒），默认 300 秒（5 分钟）。

### 自定义指令（`custom_commands`）
JSON 数组格式，每个指令包含以下字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | string | 指令名称，触发关键词 |
| `aliases` | string[] | 可选别名列表 |
| `response` | string | 机器人回复内容 |
| `action` | string | 内置动作（`switch_pro` / `switch_flash` / `show_model` / `reload_commands`） |
| `mode` | string | 发送模式（me / raw），不填则使用全局设置 |
| `admin_only` | boolean | 是否仅管理员可用 |

示例：

```json
[
  {
    "name": "帮助",
    "response": "输入 @bot 指令名 查看详情",
    "admin_only": false
  },
  {
    "name": "切换Pro",
    "action": "switch_pro",
    "admin_only": true
  }
]
```

## 7. 运行时开关

这三个开关位于实例控制台的顶部工具栏，无需进入配置页面：

- **回复** (`enable_reply`) — 是否响应 @机器人的消息。关闭后机器人将无视所有 @消息
- **感知** (`enable_auto_comment`) — 是否启用氛围评论（自动对聊天内容做评论）
- **日志** (`enable_logging`) — 是否记录聊天日志到文件

> 开关状态会自动持久化，机器人重启后生效。

## 8. 实例管理

### 新建
点击侧边栏的 **新建实例** 按钮，输入名称即可创建。

### 删除
在实例的右键菜单中选择 **删除**。删除不可撤销。

### 复制
在实例的右键菜单中选择 **复制实例**，输入新实例名称。新实例将继承原实例除 API Key 外的所有配置。

### 重命名
在实例的右键菜单中选择 **重命名**。

### 置顶 / 取消置顶
在实例的右键菜单中选择 **置顶**，可将常用实例固定在实例列表顶部。

### 导入配置
在新建实例对话框中，可以选择一个 JSON 文件导入配置。
> 导出的配置文件中 API Key 会被自动清空，需要手动填入。

### 导出配置
在实例的右键菜单中选择 **导出配置**，会下载一个 JSON 文件。
> API Key 在导出时会自动被移除，确保安全。

---

## 9. 日志

### 日志目录
日志文件保存在实例文件夹下的 `logs` 目录（可在网络设置中更改）。

### 同步日志
点击顶部工具栏的 **同步日志** 按钮，会执行以下操作：
1. 将总 TXT 日志合并生成 `chat_log.html`
2. 按日期拆分生成独立的日志文件
3. 显示同步的行数

### 打开日志文件夹
在网络设置的 **日志保存路径** 旁有一个文件夹图标按钮，点击可直接打开系统文件管理器定位到日志目录。

## 10. 常见问题

### WebSocket 连不上怎么办？
1. 确认 Minecraft 服务器正在运行
2. 确认 WebSocket 插件已安装并在正确的端口监听
3. 检查 **WS URL** 配置是否正确（默认 `ws://127.0.0.1:8080`）
4. 检查是否有防火墙阻止了连接

### AI 长时间无响应？
1. 检查实例控制台的日志区，是否有错误信息
2. 查看 `last_error`（在控制台日志中搜索"AI 调用失败"）
3. 检查 API Key 是否有效、余额是否充足
4. 尝试降低 `max_tokens` 或 `max_history` 减少 API 耗时

### 消息发不出去或格式不对？
- **raw 模式**下以 `/` 开头的文本会被游戏解释为指令
- 包含非法字符（如 `§`(仅Java版)，U+00A7）的消息会导致被踢出游戏
- 系统会自动过滤非法字符（`sanitize_reply` / `clean_for_minecraft`）
- 超过 `max_command_bytes` 的消息会被自动截断

### 配置修改后没生效？
- 大部分配置需要 **先停止再启动** 机器人才能生效
- 运行时开关（回复 / 感知 / 日志）可在运行中热切换，无需重启
- 保存配置后如果启动报错，检查控制台日志中的错误提示
